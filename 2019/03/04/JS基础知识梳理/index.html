变量类型

js是个弱类型脚本语言，意味着它定义变量时，不需要定义变量类型就可以用，（我在学Java的时候，定义一个变量需要加类型，比如定义数字类型 int num =8， 而js直接就可以 var num =8）

ECMAScript 中定义了 6 种原始类型：

  • Boolean

  • String

  • Number

  • Null

  • Undefined

  • Symbol（ES6 新定义）

    ！！原始类型不包括Object

值类型和引用类型

var a =2;

a就是值类型

var b={x:2}

b就是引用类型

如果var c =b，那么c也是个引用类型，并且与b指向的是同一个内存地址，修改b的时候c也会跟着修改

Number类型的a是按值传递的，而Object类型的b是按共享传递的。

JS 中这种设计的原因是：按值传递的类型，复制一份存入栈内存，这类类型一般不占用太多内存，而且按值传递保证了其访问速度。按共享传递的类型，是复制其引用，而不是整个复制其值（C 语言中的指针），保证过大的对象等不会因为不停复制内容而造成内存的浪费。

原型和原型链

所有的引用类型（数组、对象、函数），都具有对象特性，即可自由扩展属性

[^左边是谷歌，右边是火狐]:

所有的引用类型（数组、对象、函数），都有一个__proto__属性，属性值是一个普通的对象

[^左边是谷歌，右边是火狐]:

所有的函数，都有一个prototype属性，属性值也是一个普通的对象

所有的引用类型（数组、对象、函数），__proto__属性值指向它的构造函数的prototype属性值

当试图得到一个对象的某个属性时，如果这个对象本身没有这个属性，那么会去它的__proto__（即它的构造函数的prototype）中寻找

总结

Object和Function都作为JS的自带函数，凡是通过new Function创建的对象都是函数对象，函数对象可以创建普通对象，其他都是普通对象（通常通过Object创建），Object和Function都既是函数也是对象。每一个函数对象都有一个prototype属性，但是普通对象是没有的，每个对象都有一个名为proto_的内部属性，指向它所对应的构造函数的原型对象，原型链基于_proto;

上面所说的函数对象可以创建普通对象，而普通对象 有proto这个属性，并且指向它所对应的构造函数的原型对象

个人认为，原型就是一个属性，原型对象就是在这个属性里的对象集合，我们可以用原型添加新的方法，也就是添加到原型对象中，

原型链就是在js在找方法的时候找不到，就会去上一层找，上面图显示了，b.__proto__就是fn.prototype，如果方法没有在b.proto里找到，那就到b.proto.proto 也就是fn.prototype.proto, 而fn.prototype.proto等于的 是Object.prototype,那就会在这里找，这里也找不到最后就返回null,因为Object.prototype.proto是null, 整个原型链就是通过proto连接的，

闭包

闭包简单理解就是：能够读取其他函数内部变量的函数。

优点就是

1.保护函数内的变量安全,加强了封装性
2.在内存中维持一个变量(用的太多就变成了缺点，占内存) ，延缓数据

同步和异步

同步和异步跟js运行机制有关系，因为它是单线程，就好比排队上一个厕所，前面有人裤子解不开，那后面排队上厕所的人不能干等着他呀，就让后面的人先上，裤子解不开的那个等会儿上，，这就叫异步，而同步就是解不开裤子那个人上不完后面的谁也别上

代码比较下

异步

console.log(100) setTimeout(function () { console.log(200)//解不开裤子的人 }, 1000) console.log(300)//排在他后面的人

同步

console.log(100)
alert(200) // 不点一下弹出来的200，后面的300就不会打印出来，阻挡了后面的运行
console.log(300)

箭头函数

箭头函数是 ES6 中新的函数定义形式，function name(arg1, arg2) {...}可以使用(arg1, arg2) => {...}来定义

箭头函数有两个优点，第一个就是代码量少了，第二个就是this的指向不会指向全局变量。
